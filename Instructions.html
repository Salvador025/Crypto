Instrucciones para la configuración de la blockchain

1- Clonar el repositorio
• Clonar el repositorio
• Crear un entorno virtual: py -3 -m venv .venv (git bash)
• Instalar dependencias: pip install -r requirements.txt

2- Abrir terminales
• Abrir 5 terminales en las cuales se va a correr los diferentes tipos de usuarios y el servidor
• En las 5 terminales iniciar el entorno virtual: source .venv/Scripts/Activate (git bash)
• Entrar a la carpeta local_test en cada terminal

3- Prender servidor
• Ejecutar el código de server.py: python server.py
• Se debe tener el virtual activado y estar en local_test para que funcione

4- Conectar nodos
• En las otras 4 terminales ejecutar user_5000.py, user_5001.py, miner_5002.py, miner_5003.py
• Cada user y miner va a pedir una private key. NO SE PUEDE REPETIR entre ellos la private key. Cada uno debe tener una diferente.
• No es necesario conectar todos a la vez. Para probar de mejor manera la forma en la que se conecta, puedes conectar los dos users, hacer algunas transacciones y después conectar los miners.

5- Hacer acciones
• Los users y miners tienen 5 acciones en común:
i. Print blockchain: regresa toda la blockchain
ii. Create transaction: Crea una transacción y actualiza la red sobre esta acción. Esta acción requiere de una public key de otro usuario y una cantidad para mandar.
iii. Get balance: regresa el balance del user o miner
iv. Get public key: regresa la public key del user o miner
v. Exit: detiene el programa
• Los miners tienen otra acción:
i. Mine block: mina un bloque y actualiza a la red sobre esta acción

6- Si se detiene el programa se puede repetir el paso tres con una nueva clave para simular que un nuevo user o miner se une a la red
